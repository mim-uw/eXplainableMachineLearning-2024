The aim of Homework 1 is to train a set of differently structured predictive models and evaluate their performance on a selected dataset with a binary classification task. The behaviour of these models will be explained in future homeworks.

In terms of data used in this Homework, I chose the churn dataset. A short summary of this dataset is included below.

Imbalance ratio: 6.07
Number of rows: 5000
Number of columns: 8
Source: OpenML-CC18

The binary classification task in this dataset is based on predicting whether or not the customer churned. I have chosen this dataset as it comprises of a small and interpretable number of features, has a proper balance between the number of observations (5000) and predictive variables (8), and is not extremely imbalanced.

To show how the performance in a binary classification task can depend on model's complexity, I decided to test:

  • logistic regression, a linear model with low complexity, often used as baseline in the literature
  • random forest, an ensemble of decision tree classifiers that exhibits great performance on many difficult tasks
  • multilayer perceptron, a simple neural network with two hidden layers of 32 neurons and relu activations that often performs on-par with many other complex models

Each model's input is properly preprocessed, i.e. logistic regression and multilayer perceptron subtract the mean from each feature and divide it by its standard deviation (estimated on the training set in each fold separately).

In addition, I have also included TabPFN - a prior-fitted network trained on a very large dataset of synthetic examples and uses the Transformer architecture.

To make sure that the performance comparison is fair, I have decided to use 5-fold crossvalidation as the evaluation procedure (with each split done in a stratified fashion to account for the imbalanced classes). In terms of metrics, I have included ROC AUC as it is a standard choice when the goal is to evaluate model's performance in general (regardless of the threshold chosen on its scores), and PR AUC (Precision-Recall) to, once again, account for the imbalance, as ROC AUC is mostly insensitive to whether the classes are balanced or not. The values of these metrics were averaged over the 5 folds used in crossvalidation. The results are presented below in the form of a bar plot with an estimated standard deviation over folds and in a purely numerical fashion to make differentiating between the models easier.

As expected, logistic regression is the worst performing model, both in terms of ROC AUC and PR AUC. At around 0.66 ROC AUC (std: ~0.02) and 0.44 PR AUC (std: ~0.03), it can be considered as a fair baseline, though the most unstable in terms of PR AUC, which is more important due to class imbalance. Note that a random model on the churn dataset should perform at around 0.16 PR AUC, so even the simplest logistic regression exhibits some predictive power in this task. Random forest and multilayer perceptron perform on-par in terms of ROC AUC (0.74 vs 0.74, std: ~0.01 vs ~0.02) and PR AUC (0.45 vs 0.48, std: ~0.02 vs ~0.02) and, as expected, seem to generally perform better than logistic regression, although not with a big margin. Considering the size and complexity of TabPFN, it should not be surprising that it is a best performing model in terms of both metrics (ROC AUC: 0.76, std: ~0.02, PR AUC: 0.5, std: ~0.02).

In summary, the predictive models performed according to the initial expectations. Although they achieve high values of ROC AUC, PR AUC indicates that they are far from perfect in this difficult, imbalanced task.

def train_models_with_stratifed_cv():
    dataset_npy = DATASET.values
    x, y = dataset_npy[:, :-1], dataset_npy[:, -1]
    stratif_cv = StratifiedKFold(n_splits = N_FOLDS)
    results = {
        fold_id: {
            model_name: {metric_name: None} for metric_name in METRICS.keys() 
            for model_name in MODELS.keys()} for fold_id in range(N_FOLDS)}
    for fold_id, (train_ids, val_ids) in enumerate(stratif_cv.split(x, y)):
        print(f'Fold: {fold_id}')
        x_train, x_val = x[train_ids], x[val_ids]
        y_train, y_val = y[train_ids], y[val_ids]
        for model_name, model in MODELS.items():
            print(f'Training {model_name}')
            if model_name == 'tabpfn':
                model.fit(x_train, y_train, overwrite_warning = True)
            else:
                model.fit(x_train, y_train)
            print(f'Validation')
            y_pred = model.predict_proba(x_val)[:, -1]
            for metric_name, metric in METRICS.items():
                results[fold_id][model_name][metric_name] = metric(y_val, y_pred)
    print('Finished')
    return results

def make_summary_plot():
        fig, axs = plt.subplots(1, len(METRICS))

        metrics_aggr = {}

        for metric_id, metric_name in enumerate(METRICS.keys()):
                names = results[fold_id].keys()
                values = [np.mean([results[fold_id][model_name][metric_name] 
                                for fold_id in range(N_FOLDS)])
                                for model_name in MODELS.keys()]
                errors = [np.std([results[fold_id][model_name][metric_name] 
                                for fold_id in range(N_FOLDS)])
                                for model_name in MODELS.keys()]
                metrics_aggr[metric_name] = {'name': names, 'mean': values, 'std': errors}
                ax = axs[metric_id]
                ax.bar(names, values, yerr = errors)
                ax.set_ylim(0, 1)
                ax.tick_params(axis = 'x', rotation = 90)
                ax.set_title(metric_name)
                ax.grid()
        plt.tight_layout()
        plt.show()
make_summary_plot()

def print_metric_values():
    for metric_name in metrics_aggr.keys():
        print(metric_name)
        for k in metrics_aggr[metric_name].keys():
            print(f'{k} {list(metrics_aggr[metric_name][k])}')
        print('\n')
