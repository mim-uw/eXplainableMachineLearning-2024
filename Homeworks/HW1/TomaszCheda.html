HOMEWORK 1¶

Tomasz Cheda

I chose the phoneme dataset and developed three models, plus TabPFN. I performed a short EDA, attempted hyperparameter search, and reported the final results. For ease of reproducibility, random operations are seeded wherever feasible.

Dataset¶

Phoneme is a 1993 dataset containing sound properties of vowels as pronounced in different syllables. Each vowel in each context yields three samples, corresponding to different moments in time during pronunciation. The class label is whether a vowel is nasal (0) or oral (1). Details are available at https://www.openml.org/search?type=data&sort=runs&id=1489&status=active

There are 5 continuous numerical features plus the class feature, and 5404 samples. There are no missing values. The data is already near zero-mean and unit variance.

There are 3818 nasal and 1586 oral samples, yielding a 2.41 imbalance ratio. A dummy classifier would result in 70% accuracy

The features appear to be roughly normally distributed

Some features are weakly correlated

The dataset was split 90/10 train/test. For TabPFN, the training set was cropped to 1000 samples as suggested by an error message (technical limitation of the model)

The three models chosen are Logistic Regression, Random Tree Classifier and XGBoost. The implementations are supplied by Scikit Learn. The models were pipelined with a StandardScaler, although this is largely superfluous due to data pretransformation.

Hyperparameter search with 5-fold CV on training set was attempted for those 3, but did not identify noticeably better parameters that the defaults. The final results are reported for the defaults.

The models were evaluated with sklearn built-in accuracy, precision, recall, F1, and AUC ROC.

For an imbalanced dataset such as this one, the most reliable measure is AUCROC. Ranking by it:

  1. Random Forest: 0.9612335500237831
  2. XGBoost: 0.9502933248771207
  3. TabPFN: 0.920469319803393
  4. Logistic Regression: 0.7979546535595371

It is worth noticing that Random Forest leads both in Recall and Precision. Logistic Regression underperforms significantly. These results, with more expressive algorithms yielding better results, suggest that trying other more powerful models may be beneficial.

from sklearn.metrics import make_scorer, accuracy_score, precision_score, recall_score, f1_score, roc_auc_score
def score_pipeline(model):
  scoring = {
      'accuracy': make_scorer(accuracy_score),
      'precision': make_scorer(precision_score),
      'recall': make_scorer(recall_score),
      'f1': make_scorer(f1_score),
      'roc_auc': make_scorer(roc_auc_score, needs_proba=True)
  }

  results = cross_validate(model, X_train, Y_train, cv=5, scoring=scoring)
  accuracy_scores = results['test_accuracy']
  precision_scores = results['test_precision']
  recall_scores = results['test_recall']
  f1_scores = results['test_f1']
  roc_auc_scores = results['test_roc_auc']
  print("Accuracy Scores:", accuracy_scores)
  print("Precision Scores:", precision_scores)
  print("Recall Scores:", recall_scores)
  print("F1 Scores:", f1_scores)
  print("AUC-ROC Scores:", roc_auc_scores)
  return results

def score_single(model, xtrain, ytrain, xtest, ytest):
  model.fit(xtrain, ytrain)
  ypred = model.predict(xtest)
  yprob = model.predict_proba(xtest)[:, 1]
  accuracy = accuracy_score(ytest, ypred)
  precision = precision_score(ytest, ypred)
  recall = recall_score(ytest, ypred)
  f1 = f1_score(ytest, ypred)
  rocauc = roc_auc_score(ytest, yprob)

  print("Accuracy Score:", accuracy)
  print("Precision Score:", precision)
  print("Recall Score:", recall)
  print("F1 Score:", f1)
  print("AUC-ROC Score:", rocauc)
  return rocauc
